Standard Link Relations

  • self: carried by all resources returned by the API. Points to the resource itself. It is also carried by all Items in Collections, pointing at the related resource.
  • profile: points to the related profile of the current resource.
  • collection: carried by Item resources, points to the related Collection.
  • edit: carried by Item resources, points to itself, specifying PUT method and the appropriate JSON schema to use for the request.

Custom Link Relations

  • items: Points to the Items Collection resource. URL: /api/items/
  • item: Points to the Item resource. URL: /api/items/{item}/
  • warehouses: Points to the Warehouse Collection resource. URL: /api/warehouses/
  • warehouse: Points to the Warehouse resource. URL: /api/warehouses/{warehouse}/
  • catalogue: Points to the Catalogue Collection resource. URL: /api/catalogue/
  • catalogue_item: Points to the Catalogue Item resource. URL: /api/catalogue/supplier/{supplier}/item/{item}/
  • stock: Points to the Stock Collection resource. URL: /api/stocks/
  • stock_item: Points to a specific Stock Item resource within a warehouse. URL: /api/stocks/{warehouse}/item/{item}/
  • stock_item_collection: Points to the collection of Stock Items for a specific item across warehouses. URL: /api/stocks/item/{item}/
  • stock_warehouse_collection: Points to the collection of Stock Items within a specific warehouse. URL: /api/stocks/warehouse/{warehouse}/
  • locations: Points to the Locations Collection resource. URL: /api/locations/
  • location: Points to the Location resource. URL: /api/locations/{location}/
